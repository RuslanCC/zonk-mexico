# 🎲 Кости — Зонк и Мексика

Адаптивное веб-приложение-компаньон для настольных игр в кости **Зонк** (Farkle / 10 000)
и **Мексика** (Mexican / Mia). Ведёт счёт, время партии, историю и подсказывает правила —
чтобы за столом не считать в уме и не спорить.

Работает офлайн, устанавливается на телефон как приложение (PWA). Без сборки, без сервера —
чистые HTML/CSS/JS.

## Возможности

- **Два режима партии**
  - *Зонк* — ручное табло, тотал у каждого игрока, индикатор чьего хода, кнопка «Зонк» (0),
    настраиваемая цель (по умолчанию 4000), победный экран. Виртуальные кости считают
    очки броска по правилам автоматически (включая малые стриты 1–5 = 500 и 2–6 = 750).
  - *Мексика* — модель на жизнях, выбывание, последний оставшийся побеждает. Виртуальные
    кости раскладываются «старший слева, младший справа». Есть необязательный трекер
    «Ставка раунда» для блеф-версии (Mia).
- **Таймер партии** — идёт с паузой, длительность сохраняется в историю.
- **История игр** — все партии хранятся в `localStorage`, с результатами и временем.
- **Статистика игроков** — партии, победы, %, средний и лучший счёт, серии побед.
- **Справочник правил** — встроенный, с таблицей комбинаций Зонка.
- **Игроки** — быстрое добавление (имя + Enter), эмодзи и цвет, правка/удаление.
- **Достижения и звуки** — весёлые бейджи + звуковые эффекты (генерируются через Web Audio,
  без аудиофайлов).
- **Тёмная/светлая тема** — запоминается.
- **Отмена хода (Undo)** на обоих табло.
- **Автосохранение партии** — перезагрузка не теряет счёт, есть кнопка «Продолжить».
- **Виртуальные кости** — бросок с анимацией, если под рукой нет настоящих.
- **Экспорт/импорт данных** (JSON) — резервная копия и перенос между устройствами.

## Запуск локально

Нужен любой статический сервер (модули ES и service worker не работают через `file://`):

```bash
cd zonk
python3 -m http.server 8000
# открыть http://localhost:8000
```

## Публикация на shared hosting

Просто скопируйте всю папку в public-каталог хостинга (например `public_html/`) по FTP.
Открывайте по **HTTPS** — это обязательно для установки PWA и работы офлайн-режима
(service worker требует HTTPS либо localhost).

## Обновление версии

При изменении файлов увеличьте номер кэша в [`sw.js`](sw.js) (константа `CACHE`,
например `kosti-v1` → `kosti-v2`) — старый кэш очистится и подтянутся новые файлы.

## Структура

```
index.html              — каркас страницы
css/styles.css          — стили и темы
js/                     — модули (ES modules, без сборки)
  app.js                — роутер, главная, настройки, тема
  store.js              — состояние в localStorage
  zonk.js / mexico.js   — логика партий
  game-common.js        — выбор игроков, финиш, победный экран
  history.js            — история и статистика
  rules.js              — справочник правил
  players.js            — управление игроками
  achievements.js       — достижения
  sound.js / confetti.js / dice.js / timer.js / ui.js — вспомогательное
manifest.webmanifest    — манифест PWA
sw.js                   — service worker (офлайн)
icons/                  — иконки приложения
```

Внешних зависимостей и CDN нет — всё работает офлайн.
